# AGENTS.md

このリポジトリで作業するエージェント向けの指針をまとめる。

## 基本方針

- 対話は日本語で行う。
- このプロジェクトでは、Slurm プラグイン開発に伴う拡張動作を、実際にジョブを投入して確認しながら調査する。
- 机上の推測だけで結論を出さず、可能な限り `sbatch` / `srun` / `salloc` などを使った実測結果で裏付ける。
- 検証時は、投入コマンド、主要な設定、観測できた結果を必ず記録できる形で残す。

## 参照資料

- プラグインの仕様・実装方針は [notes/slurm-rsc-option-impl.md](/local/w550380/repos/slurm-custom-jobsubmit-rscopt-test/notes/slurm-rsc-option-impl.md) を第一参照とする。
- `--rsc` オプションの解釈、SPANK プラグインと `job_submit` プラグインの責務分担、環境変数の扱い、競合オプションの扱いは上記ノートに従う。
- 追加の判断が必要な場合も、まずはノートの記述と矛盾しないことを確認する。

## 調査・実装の進め方

- 変更前に、関連ファイルと既存ノートを読んで前提を確認する。
- Slurm の挙動確認が目的の作業では、実際のジョブ投入結果を優先し、必要なら最小構成の再現ケースを作る。
- プラグイン実装や検証条件を変えた場合は、何を変えたか、どの条件で試したかを明確にする。
- 新しく得られた知見は `notes/` 配下に追記し、再現に必要な情報を省略しない。

## 注意点

- 既存の未コミット変更はユーザーの作業である可能性があるため、無断で巻き戻さない。
- 破壊的な操作や広範囲な設定変更は、必要性を確認してから行う。
- 実環境依存の結果については、推測と観測事実を区別して記述する。
